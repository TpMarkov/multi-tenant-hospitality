# Database Setup Guide

The "Failed to load menu" error is occurring because the backend cannot connect to a MongoDB database. Follow these steps to get everything running:

## 1. Install or Connect to MongoDB

### Option A: Local MongoDB (Recommended for Dev)
1. Download and install **MongoDB Community Server**: [https://www.mongodb.com/try/download/community](https://www.mongodb.com/try/download/community)
2. Once installed, MongoDB will run in the background on `mongodb://localhost:27017`.
3. You don't need to create a database; the app will do it automatically.

### Option B: MongoDB Atlas (Cloud)
1. Sign up for a free account at [https://www.mongodb.com/cloud/atlas](https://www.mongodb.com/cloud/atlas).
2. Create a "Shared" cluster (free).
3. Under "Network Access", allow your current IP address.
4. Under "Database Access", create a user (e.g., `admin`) and password.
5. Get your connection string (it looks like `mongodb+srv://...`).
6. Update the `MONGODB_URI` in your `server/.env` file with this string.

---

## 2. Seed Sample Data

Once MongoDB is running, you need some data to display in the menu.

1. Open your terminal in the `server` folder.
2. Run the following command:
   ```bash
   npm run seed
   ```
3. This will create a sample hotel, room, and menu items.

---

## 3. Launching the App

1. Make sure the backend is running: `cd server && npm run dev`.
2. Make sure the frontend is running: `cd client && npm run dev`.
3. The seeding script will print a unique link (e.g., `http://localhost:3000/property/6617.../room/101`).
4. **Copy and paste that exact link** into your browser.

---

## Why "sample-room-1" didn't work initially?
MongoDB uses unique 24-character IDs (like `6617afde...`) for security and multi-tenancy. The "sample" link was just a placeholder. Running `npm run seed` will provide you with a valid real-world link generated by your own database.
